CSP Management System Agent Login


How it works ?
In this project, the CSP Management System, the agent can login to the portal by entering correct username and password. Agent can open account of any customer by entering customer’s personal details, opening balance, document details for verification. Agent can login customer’s account by account number and OTP for view user bank details, deposit amount, withdraw amount, view transaction, and e-kyc verification. The agent must submit mobile and OTP in order to gain access to customer’s account

Account Opening
The system allows an agent to open a bank account for a new customer. The agent fills out a registration form with the required details, verifies the entered information, and can make edits if needed. The form undergoes verification and validation processes, checking for completeness and valid data. If any errors are found, appropriate error messages are displayed. Once the verification and validation are completed, the agent verifies the customer's mobile number linked to their Aadhaar number through OTP verification. After successful verification, the data is saved in the database, and a pop-up message confirms the successful creation of the account.

Deposit Amount
To deposit money into a customer's account, the agent enters the account number and linked mobile number to gain access. The program verifies the information and sends an OTP for authentication. If the OTP is correct, the agent can access the account and deposit the desired amount. The transaction is recorded as a transaction history, and the customer's account balance is updated in the database.

withdraw Amount
The agent can access a customer's account by entering the account number and linked mobile number. A correct OTP allows access, while a wrong OTP restricts it. The agent can withdraw money from the customer's account after verifying an OTP sent to the customer. The transaction is recorded in the database, updating the account balance.

Check Bank Balance
The agent gains access to the customer's account by entering the account number and linked mobile number, followed by verifying an OTP. With the correct OTP, the agent can view the customer's bank balance, while an incorrect OTP restricts access.

Money Transfer
The agent can transfer money from the customer's account by gaining access through the account number and linked mobile number. After verifying the OTP, the agent can enter the receiver's account number and IFSC code to complete the transfer. The transaction is recorded in the database, updating the account balance.

Mini Statement
The agent is able to retrieve the bank statement of an account holder by inputting the accurate account number and IFSC code. To access the mini statement of the account holder, the agent is required to authenticate the One-Time Password (OTP) as a validation measure. This ensures the security and privacy of the account holder's financial information.
